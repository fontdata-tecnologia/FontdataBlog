# Feature: Feed RSS para o blog

**Data:** 2026-05-20  
**Autor:** Miguel Viana  

---

## Objetivo

Adicionar um feed RSS ao blog para que leitores possam acompanhar novos artigos via leitores de RSS (Feedly, Inoreader, etc.) e para melhorar a descoberta do conteúdo por agregadores e buscadores.

---

## O que foi feito

### 1. Rota de geração do feed RSS (`app/feed.xml/route.ts`)

Criada uma route handler no App Router do Next.js que responde em `GET /feed.xml` e retorna XML válido no formato RSS 2.0.

**Detalhes técnicos:**

- Busca os 50 posts mais recentes com `status = 'published'`, ordenados por `published_at DESC`
- Faz join com as tabelas de categorias e tags via junction tables (`post_categories`, `post_tags`)
- Usa o namespace `content:encoded` para incluir o HTML completo do post dentro de blocos CDATA
- Usa o namespace `atom:link` com `rel="self"` para auto-referência do feed
- Imagem de capa (`cover_image`) é incluída como `<enclosure>` quando existe
- Categorias e tags aparecem como elementos `<category>`
- O excerpt do post é usado como `<description>`, com fallback para as primeiras 300 caracteres do conteúdo sem HTML
- O idioma do feed é `pt-BR`
- Headers de cache: `s-maxage=600` (10 minutos), `stale-while-revalidate=300`
- Content-Type: `application/xml; charset=utf-8`

**Estrutura XML gerada:**

```xml
<?xml version="1.0" encoding="UTF-8"?>
<rss version="2.0"
  xmlns:content="http://purl.org/rss/1.0/modules/content/"
  xmlns:atom="http://www.w3.org/2005/Atom">
  <channel>
    <title>Nome do Blog</title>
    <link>https://dominio.com</link>
    <description>Descrição do blog</description>
    <language>pt-BR</language>
    <lastBuildDate>Wed, 20 May 2026 14:00:00 GMT</lastBuildDate>
    <atom:link href="https://dominio.com/feed.xml" rel="self" type="application/rss+xml"/>
    <item>
      <title>Título do Post</title>
      <link>https://dominio.com/slug-do-post</link>
      <guid isPermaLink="true">https://dominio.com/slug-do-post</guid>
      <description>Resumo do post...</description>
      <pubDate>Wed, 20 May 2026 12:00:00 GMT</pubDate>
      <enclosure url="https://..." type="image/jpeg" length="0"/>
      <category>Nome da Categoria</category>
      <category>Nome da Tag</category>
      <content:encoded><![CDATA[<p>Conteúdo HTML completo...</p>]]></content:encoded>
    </item>
  </channel>
</rss>
```

### 2. Auto-descoberta no layout público (`app/(public)/layout.tsx`)

Adicionada a função `generateMetadata()` ao layout público com a propriedade `alternates.types` apontando para `/feed.xml`. Isso faz com que o Next.js inclua automaticamente a tag `<link rel="alternate" type="application/rss+xml">` no `<head>` de todas as páginas públicas, permitindo que browsers e leitores de RSS descubram o feed automaticamente.

```typescript
export async function generateMetadata(): Promise<Metadata> {
  const { company } = await getSettings()
  const blogName = company.blog_name || process.env.NEXT_PUBLIC_BLOG_NAME || 'Blog'
  const baseUrl = process.env.NEXT_PUBLIC_APP_URL || 'http://localhost:3000'
  return {
    title: { default: blogName, template: `%s | ${blogName}` },
    alternates: { types: { 'application/rss+xml': `${baseUrl}/feed.xml` } },
  }
}
```

## Como testar

1. Inicie o servidor: `npm run dev`
2. Acesse `http://localhost:3000/feed.xml` — deve retornar XML válido com os posts publicados
3. Em qualquer página pública, visualize o código-fonte e confirme a presença de `<link rel="alternate" type="application/rss+xml" href="...">` no `<head>`
4. Valide o feed em [validator.w3.org/feed](https://validator.w3.org/feed/)

## Prompt para recriar em outro blog

> **Contexto:** Meu blog usa Next.js (App Router), TypeScript, Tailwind CSS, Drizzle ORM e PostgreSQL (Supabase). Os posts ficam na tabela `posts` com os campos: `id`, `title`, `slug`, `content` (HTML), `excerpt`, `cover_image`, `status` (`draft`/`published`), `published_at`, `created_at`, `updated_at`. Categorias e tags usam junction tables (`post_categories`, `post_tags`). As configurações do site (nome do blog, descrição, etc.) vêm de uma função `getSettings()` que retorna `{ company: { blog_name, blog_description, ... } }`. A URL base está em `NEXT_PUBLIC_APP_URL`.
>
> **Tarefa:** Crie uma página de feed RSS para o blog seguindo estes requisitos:
>
> 1. Crie um route handler em `app/feed.xml/route.ts` que retorne XML no formato RSS 2.0 com:
>    - Namespace `content:encoded` para conteúdo HTML completo do post
>    - Namespace `atom:link` com `rel="self"` apontando para `{baseUrl}/feed.xml`
>    - Até 50 posts publicados (`status = 'published'`), ordenados por `published_at` DESC
>    - Cada `<item>` deve conter: `<title>`, `<link>`, `<guid>`, `<description>` (excerpt ou fallback de texto sem HTML), `<pubDate>`, categorias e tags como `<category>`, imagem de capa como `<enclosure>`, e o HTML do post em `<content:encoded>` dentro de CDATA
>    - O `<channel>` deve ter: nome do blog, URL base, descrição, idioma `pt-BR`, `<lastBuildDate>` e o `<atom:link rel="self">`
>    - Headers de resposta: `Content-Type: application/xml; charset=utf-8` e `Cache-Control: public, s-maxage=600, stale-while-revalidate=300`
>    - Escape correto de caracteres XML especiais (`&`, `<`, `>`, `"`, `'`)
>
> 2. No layout público (`app/(public)/layout.tsx`), adicione uma função `generateMetadata()` que retorne `alternates.types` com a chave `'application/rss+xml'` apontando para `{baseUrl}/feed.xml`, para que browsers descubram o feed automaticamente via tag `<link>` no `<head>`.
>
> **Importante:** Use as queries Drizzle ORM existentes como referência (a rota `/api/posts/route.ts` já faz o join com categorias e tags). Busque os dados diretamente do banco, sem passar pela API HTTP. Use `export const dynamic = 'force-dynamic'` no route handler.
