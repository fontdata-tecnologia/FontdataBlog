# #16 — Gestão de Newsletter

**Data:** 2026-05-20
**Stack:** Next.js 14 App Router · Drizzle ORM · PostgreSQL

---

## O que foi implementado

Sistema completo de captura e gestão de inscrições de newsletter. Quando habilitado, exibe um card de inscrição em todas as páginas públicas do blog. A área administrativa permite configurar o card e gerenciar os inscritos.

## Schema — tabela `newsletter_subscribers`

```sql
CREATE TABLE newsletter_subscribers (
  id              SERIAL PRIMARY KEY,
  email           TEXT UNIQUE NOT NULL,
  status          TEXT NOT NULL DEFAULT 'active',   -- 'active' | 'unsubscribed'
  subscribed_at   TIMESTAMP NOT NULL DEFAULT now(),
  unsubscribed_at TIMESTAMP
);
CREATE INDEX newsletter_email_idx  ON newsletter_subscribers (email);
CREATE INDEX newsletter_status_idx ON newsletter_subscribers (status);
```

---

## Configuração — chave `newsletter_config` em `site_settings`

```json
{
  "enabled": false,
  "title": "Fique por dentro das novidades",
  "subtitle": "Receba os melhores artigos diretamente no seu e-mail."
}
```

Salva/atualiza via `PUT /api/admin/settings` com body `{ newsletter: { enabled, title, subtitle } }`.

---

## Fluxo completo

```
Usuário público
  └─> vê o card na página (se newsletter.enabled = true)
  └─> digita e-mail e clica "Inscrever-se"
  └─> POST /api/newsletter  { email }
        ├─ e-mail novo       → INSERT newsletter_subscribers
        ├─ e-mail cancelado  → UPDATE status = 'active'
        └─ e-mail ativo      → 409 Conflict

Admin
  └─> /admin/newsletter
        ├─ toggle habilitar/desabilitar card
        ├─ editar título e subtítulo
        ├─ salvar → PUT /api/admin/settings { newsletter: {...} }
        ├─ ver lista de inscritos (ativos + cancelados)
        └─ cancelar inscrição → DELETE /api/admin/newsletter?id=X
                                  → UPDATE status = 'unsubscribed'
```

## Decisões de design

- **Card no layout, não nas pages** — o `NewsletterSection` foi movido para `app/(public)/layout.tsx` para aparecer em todas as rotas sem duplicar código em cada template.
- **Cores do tema via CSS vars** — o card usa `var(--color-primary)` no fundo e `var(--color-secondary)` no botão, respeitando automaticamente o tema ativo.
- **Reativação de inscrição** — se alguém que cancelou tentar se inscrever novamente, o status volta para `active` em vez de retornar erro.
- **Configuração em `site_settings`** — segue o padrão existente do projeto (mesmo pattern que `company_info`, `theme_colors`, etc.), sem criar tabela separada para config.
- **Apenas captura de e-mails** — nenhum envio de e-mail implementado. Os inscritos ficam armazenados para uso futuro.

## Prompt para replicar em outro projeto

Copie o prompt abaixo para o Claude Code (ou outro agente) em um projeto Next.js 14 com Drizzle ORM + PostgreSQL:

```
Implemente um sistema de newsletter para este blog Next.js. Requisitos:

1. BANCO DE DADOS
   - Crie uma tabela `newsletter_subscribers` com: id (serial pk), email (text unique not null),
     status (text, enum 'active'|'unsubscribed', default 'active'), subscribed_at (timestamp default now()),
     unsubscribed_at (timestamp nullable).
   - Crie índices em email e status.

2. CONFIGURAÇÃO
   - Armazene a config da newsletter em site_settings com chave 'newsletter_config' como JSON:
     { enabled: boolean, title: string, subtitle: string }
   - Valores padrão: enabled=false, title='Fique por dentro das novidades',
     subtitle='Receba os melhores artigos diretamente no seu e-mail.'
   - Exponha a config no getSettings() como campo `newsletter: NewsletterConfig`.

3. API PÚBLICA
   - POST /api/newsletter — aceita { email }, valida, insere ou reativa inscrição.
     Retorna 409 se já estiver ativo.

4. API ADMIN (protegida por middleware)
   - GET /api/admin/newsletter — retorna todos os inscritos ordenados por subscribed_at desc.
   - DELETE /api/admin/newsletter?id=X — muda status para 'unsubscribed'.
   - PUT /api/admin/settings aceita campo `newsletter: { enabled?, title?, subtitle? }` e persiste.

5. COMPONENTE PÚBLICO
   - Componente `NewsletterSection` (client component) com props title e subtitle.
   - Faz POST /api/newsletter ao submeter.
   - Estados: idle, loading, success, error.
   - Usa cores do tema via CSS vars (--color-primary para fundo, --color-secondary para botão).
   - Aparece em TODAS as páginas: injete no layout público antes do footer,
     condicionado a newsletter.enabled === true.

6. PÁGINA ADMIN (/admin/newsletter)
   - Toggle para habilitar/desabilitar o card.
   - Campos para editar título e subtítulo com botão salvar.
   - Tabela de inscritos: e-mail, data de inscrição, badge de status (Ativo/Cancelado),
     botão "Cancelar inscrição" para inscritos ativos.
   - Adicione item "Newsletter" no menu de navegação admin.

Não implemente envio de e-mails — apenas captura. Siga os padrões de código já existentes no projeto.
```
